Lógica Booleana
Como sabes, las señales digitales binarias pueden tomar sólo dos posibles valores 0V o 5V. Una variable binaria es muy similar, ya que sólo puede tomar valores de VERDADERO o FALSO, normalmente lo representamos como 1 o 0. El Algebra Booleana es la parte del algebra que trabaja con variables binarias, esto es la que define las reglas y la representaión de variables binarias. Fue formalizado por George Boole, que introdujo las relaciones equivalentes al álgebra numérica.
Hay tres operaciones básicas: Y o AND (conjunción), O o OR (disjunción) and NO o NEG (negación). Una operación Y acepta dos variables binarias y devuelve 1 si, y sólo si, las dos son 1. Una operación O acepta dos variables y devuelve 1 cuando cualquier de las variables es 1. Una operación NO acepta sólo un valor y devuelve el valor contrario, estos es 0 si es 1 y 1 si es 0 (evidentemente la negación de la negación es equivalente al valor original).
Hay otras operaciones útiles, como por ejemplo la XOR o O EXCLUSIVA. Esta operación acepta dos variables binarias y devuelve 0 si son iguales y 1 si son diferentes.

Además, hay idendidades cuando combinamos algunas de estas operaciones básicas. Se conocen como la Ley de De Morgan:

X AND Y=NOT(NOT(X) OR NOT(Y))

X OR Y=NOT(NOT(X) AND NOT(Y))

Bits y Números
Los ordenadores no entienden los números de la forma en la que nosotros lo hacemos. Tampoco los microcontroladores como Arduino o ESP. Ellos sólo entienden bits. Un bit es la unidad básica de la información. Como quizás sepas, sólo puede almacenar dos valores: 0 o 1. Así pues, para poder representar números más grandes se requieren grupos de bits, de la misma forma que al usar la representación decimal con los dígitos del 0 al 9. Cada vez que necesitamos representar un número más grande, añadimos un dígito más a la izquierda y el resto los ponemos a 0. Así pues, del 9, añadirmos un 1 a la izquierda y el 9 lo transformamos en 0 para crear el 10. En la representación binaria hacemos exactamente lo mismo, para representar un número más grande añadimos un dígito con un 1 a la izquierda y el resto los ponemos a 0. En los microcontroladores, cuando usamos un número que se almacena en memoria, esto suele requerir una cantidad de bits necesaria para poder almacenarlo. Típicamente, almacenamos los datos grupos de 8-bits (byte), 16-bits (palabra o entero sin signo) o 32-bits (doble palabra o entero largo sin signo). Así pues, un byte puede almacenar hasta 2^8 números, esto es del 0 al 255. Una palabra puede representar los números desde el 0 al 65535 y una doble palabra puede representar los números desde el 0 al 4294967295. Sí, puede parecer mucho, pero en realidad a veces necesitamos almacenar números más grandes. A veces, también necesitamos representar números negativos. La forma en la que suele hacerse esto es utilizar el bit más significativo (MSB), el de más a la izquierda, para representar el signo menos.
Bitwise operations
Now you know a little bit about numbers and bits, you might be interested in knowing that bitwise operations can be done with numbers. This might be useful to check whether or not a bit is 1 or 0, because this bit is representing some kind of useful information for a given data. Yes, we know this is going to deep on how data is represented, but believe us when we say that there are many applications and reasons for that. Just an example, an easy way to check if your data being transmitted propperly is to use an error check system. One of the simplest ways of doing that is to perform an XOR operation on sent data and then send a last byte with the result of this XOR operation. When the receiver receives the data, uses the same XOR operation on data and if the resuts match, then it is very likely that the data is correct and has not been corrupted.

Basic Exercises
  1. Set LED when pushed

    Following the ideas of basic exercise 3 of Bliking LEDs Exercise, we want to set the LED connected to pin D12 when we push the push-button (connected to pin D2).

    Hint:

    Use the 'DigitalRead' instruction to get the value and its returned value to the 'not' instruction to negate its value. The resulting value must be used as the state input of the 'DigitalWrite' instruction.

  2. AND operation

    Implement Boolean operation using the two push-buttons (connected to pins D2 and D3) and the red LED (connected to pin D12). When both of them are pushed, the pin of the LED must be set to HIGH.

    Hint:

    Considering push-button D2 as PB1 and push-button D3 as PB2, the resulting operation for setting the LED state is LED=NOT(PB1) AND NOT(PB2).

  3. Equivalent AND operation

    Now, using De Morgan's Law, find an equivalent implementation of previous exercise but with less operations.

    Hint:

    In this case, you need to implement LED=NOT(PB1 OR PB2).

Intermediate Exercises
  1. AND and OR Operations

    Implement binary operations using the button switches (connected to pins D2 and D3) and the red and blue LEDs (connected to pin D12 and D13). The aim is to implement the following logic operations: RED=¬(SW1∧SW2) (this is equivalent to AND Operation), BLUE=¬(SW1∨SW2) (this is equivalent to OR operation). All required Electronics are already integrated in the Multisensorshield, the wemos Microcontroller just has to be inserted.

  2. Even or Odd (Bitwise And)

    Turn on LED D12 whenever we introduce an even number on the USB serial console. Turn on LED D13 when it's odd.

    Hint:

    Use a bitwise 'and' to mask the number with '1'. If the result is greater than zero then the number is odd.

  3. Comparison

    Read the analog input A0 corresponding to the potentiometer position. Divide the analog read range into 4 sub-ranges and turn on LEDs D12 and D13 based on the binary codification of each range.

    Hint:

    If the number is between 0 and 1023, then both LEDs must be off, if the value is between 1024 and 2047 then LED D12 must be on and D13 off, if the value is between 2048 and 3071 then LED D13 must be on and D12 off. Otherwise, both LEDs must be on.
