Temperatura y Humedad Digital (DHT11)

Aprende a leer los datos de temperatura y humedad del sensor DHT11 y establece una alarma si las temperaturas o humedades son demasiado altas o demasiado bajas.

Material requerido.
  • Una placa Arduino o ESP, es decir: Arduino Uno , Arduino Nano , Wemos D1 R2 o Wemos D1 R32 , NodeMCU , etc.
  • Una tarjeta multisensor o un módulo de sensor DHT11 .

Ejercicios básicos
  1. Leer temperatura

    Lee la temperatura con el sensor DHT11 conectado al pin D4 cada 5 segundos e imprime el resultado en el monitor en serie.

    Pista:

    Utiliza las instrucciones 'Sensor de temperatura' y 'Espera (s)'.

     
  2. Leer humedad

    Ahora, lee la humedad con el sensor DHT11 conectado al pin D4 cada 5 segundos e imprime el resultado en el monitor en serie

    Pista:

    Utiliza las instrucciones 'Sensor de humedad' y 'Espera (s)'.

     
  3. Temperatura de lectura cada 5000 ms

    Ahora, modifiqca el primer ejercicio y lee la temperatura cada 5000 ms utilizando la instrucción 'Cada'. Incluye también un texto para imprimir junto con el valor de temperatura: 'Temperature (Celsius):'

    Pista:

    Utiliza las instrucciones 'Sensor de temperatura' y 'Cada'.

     
  4. Leer la humedad cada 5000 ms

    Ahora, modifica el primer ejercicio y lee la humedad cada 5000 ms utilizando la instrucción 'Cada'. Incluya también un texto para imprimir junto con el valor de humedad: 'Humedad (%):'

    Pista:

    Utilice las instrucciones 'Sensor de humedad' y 'Cada' .

     
  5. Alarma de temperatura

    Imprime un mensaje si la temperatura es muy alta o muy baja

    Pista:

    Usa la instrucción 'Alarma de temperatura'. No es necesario incluir un retardo adicional después de las instrucciones de la alarma porque internamente espera 1000 ms después de verificar los casos de alarma .

  6. Alarma de humedad

    Imprime un mensaje si la humedad es demasiado húmeda o demasiado seca

    Pista:

    Utiliza la instrucción 'Alarma de humedad'. No es necesario incluir un retardo adicional después de las instrucciones de la alarma porque internamente espera 1000 ms después de verificar los casos de alarma .

Ejercicios intermedios
  1. Leer temperatura y humedad cada 5000 ms

    Ahora lee ambos, temperatura y humedad, cada 5000 ms. .

    Pista:

    Ahora, declara dos variables 'temperature' y 'humeidity' y almacena el resultado de leer ambos valores. Incluye un retardo de 1000 ms entre las lecturas de temperatura y humedad para garantizar que el sensor funcione como se espera.

     
  2. Trazado de temperatura y humedad

    Dibuja la temperatura y la humedad cada 5000 ms y muéstrala en la consola. Usa el sensor DHT11.

     
  3. Alarmas de temperatura y humedad

    Imprime mensajes en la consola cuando la temperatura y la humedad cambien a valores incómodos, incluidos los casos 'demasiado caliente', 'temperatura perfecta', 'demasiado frío', 'demasiado húmedo', 'humedad perfecta' y 'demasiado seco'.

    Pista:

    Usa las instrucciones de alarma de temperatura y humedad. No es necesario incluir un retardo adicional entre dos instrucciones de alarma consecutivas porque internamente espera 1000 ms después de verificar los casos de alarma.
