Analog Input

An Arduino's analog inputs detailed description

Analog signal

An analog signal is a signal that can take any value between 0V and 5V (for Arduino Uno). Therefore, the main difference with respect to a digital signal is that digital signals can only take two possible values, 0V or 5V, while an analog signal could be any real number between 0V and 5V, for example 1.2V.

(From Wikipedia) An analog signal is any continuous signal for which the time varying feature (variable) of the signal is a representation of some other time varying quantity, i.e., analogous to another time varying signal.
Analog Inputs

An analog input is a signal measured by the microcontroller corresponding to an analog signal of a sensor with voltages between 0V and 5V. However, microcontrollers do not understand analog values, they can only work with digital values. Then, how is this possible? The answer is very simple, they use an analog-digital converter (ADC) to convert an analog value into an integer number.

The only thing you need to know is that it is a linear conversion process is carried out by Arduino internally and a voltage 0V is converted into 0 and a voltage of 5V is converted into 1023.

Associated with the analog-digital conversion, there's a quantization phenomena related with the ADC resolution. Arduino Uno's ADC resolution is 10 bit, that's why 5V is converted into 1023=(2^10)-1. This implies that there's information loss during the conversion process, because a real number is transformed into an integer number. In other words, you won't be able to distinguish between two analog values whose difference is smaller than 4.88mV=5V/(2^10).

Let' make it simple! Suppose that your ADC resolution is only 2bit, then, the only possible voltage range values you can distinguish are (0V-1.25V), (1.25V-2.5V), (2.5V-3.75V) and (3.75V-5V), because the voltage resolution is clearly 1.25V=5V/(2^2). The following figure illustrates this phenomena.

Coding with Facilino

In order to read the value of an analog input in Facilino we need to use the following instructions. The first instruction returns a value between 0 and MAX_ADC, being MAX_ADC=2^b-1 and b the resolution in bits of the Analog-Ditigal converter (ADC). In Arduino Uno MAX_ADC=1023, while for ESP8266 and ESP32 processors MAX_ADC=4095. The analog read instruction requires the use of the second instruction to indicate the appropriate analog pin number.
