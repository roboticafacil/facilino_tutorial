Interrupts

Interrupts are certain signals that will interrupt the current program flow so that a higher priority task or a break can be executed. microcontroller programms work normaly in a linear order, that loops upon completion. Usually most microcontrollers include different kind of interrupts to allow the user to jump between task to have a more useful and natural workflow.

Timer Interrupts

Timers are included in most microcontrollers to measure time or execute tasks periodically. Timers include registers where to store the current timer counter value. This value is updated with the microcontroller clock signal together with a prescaler (to make the update signal slower, because clock ticks are usually too fast). When the timer counter reaches pre-defined value, this triggers an interrupt so a specific task can be executed. To control the period where the interrupt is executed, you need to modify the pre-defined value.

External Interrupts

External interrupts can generate an interrupt on the work flow on specific pins. The interrupt can be triggered when the pin attached to the interrupt is set to a given level (HIGH or LOW) or at the falling or rising edges of the signal, that is, transitions between HIGH to LOW (falling) or LOW to HIGH (rising).

Coding with Facilino

In order to generate timer-based interrupts, Facilino uses a multi-task instruction that allows you to execute tasks periodically with a minimum period of 1ms using the first instruction. This instruction is a mutable instruction, which means that you can add as many tasks are desired. Caution: When working with timers, you must consider the fact that other libraries, might be using that timer too, so executing time-consuming tasks inside this instruction could generate a lot of problems and things won't work as expected. This a common issue arised due to sharing a common resource and Facilino does its best to overcome part of these situations, but there might be cases in which there might be problems.

The second instruction is used to run code at indicated interrupt signal mode (low, change, rising edge or falling edge). Again, using this type of instruction it might affect to execution of other instructions, due to the execution of the code inside the statement input. Ideally, this code should be fast to execute.
