Function and Procedures

Learn how to implement functions and procedures to resue part of your code and call it at any part of your program.

All required Electronics are already integrated in the Multisensor Shield.
Required material.
  • Arduino Uno (and compatible) or Wemos D1R2/D1R32.
  • A Multisensor Shield.
Preliminaries
Procedures are very convinient whenever you need to execute a set of instructions several times. It, generally, saves space in terms of the code require to execute a program, and it is a good programming style to use them whenever they are useful. Functions are just like procedures, but they return a value as a result of a code execution. Both, procedures and functions can have several input arguments of different types (i.e.: integers, strings, etc.). Functions have a mandatory return output argument of the indicated type.
Morse Code
Morse code is a method of transmitting text information as a series of on-off tones, lights, or clicks that can be directly understood by a skilled listener or observer without special equipment. The Morse code two types of codes: dash and dot. The dot takes one unit of base time (depends on the sender), while the dash takes three units of base time. The time (without light or sound) between consecutive dashes and dots is one unit of base time, while the time between different letters is three units of base time and between words is seven units of base time. Encryption
Encryption The Atbash cipher is a simple substitution cipher from Biblical times; it reverses the alphabet such that each letter is mapped to the letter in the same position in the reverse of the alphabet. The original implementation (ca. 500 BC) was for the Hebrew alphabet and there are Old Testament references to it. The Atbash cipher has also been associated with various forms of mysticism. In modern times, it is referred to as a reverse alphabet code

Basic Exercises
  1. Blink a LED as you wish I

    Implement a procedure that blinks the LED connected at pin D12 as many times as indicated by an input argument of type 'int' with a blinking frequency of 1Hz. Then, in the main loop, parse an integer received from the USB serial port and call the procedure to make the LED blink.

    Hint:

    Add an input argument of type 'int', let's say 'n' and use it inside the procedure to count from '1' to 'n' in a loop. Inside the loop make the LED blink, setting its state to HIGH and LOW with a delay of 500ms. To parse an integer number Intermediate Exercise 1 in USB Serial Communication.

  2. Blink a LED as you wish II

    Now, extend the previous procedure so that the delay time is also an input argument.

    Hint:

    Now, you need to add a second input argument of type 'int' and replace the constant '500' from the previous exercise with the corresponding time value variable (second input argument). Now, in the serial monitor, you need to enter two consecutive numbers every time you what to make the LED blink

  3. Sum two numbers

    Implement a function that takes two input arguments and returns the sum of them. Parse two integer numbers received from the serial monitor and call the function to perform the sum. Print back on the serial monitor the result of the sum.

    Hint:

    Create a function with two input arguments of type 'int' and return the sum of them in 'return' section. When you introduce two numbers, you may notice that the program eventually returns 0 even if you don't type anything. This is because is receiving additional characters and interpreting them as 0.

Intermediate Exercises
  1. Basic S.O.S.

    Implement two functions: dash and dot. The dash and dot functions will turn on and off blue LED (connected to pin D13). The base time for the dash and dot functions will be fixed to 200ms. Try to transmite and S.O.S message based on the table in the Preliminaries (after the message has been transmitted, you can repeat it if you make a long pause, say of 10000ms). All required Electronics are already integrated in the Multisensorshield.

    Hint:

    The sequence for the dot function is: turn on LED, wait 200ms, turn off LED and wait 200ms. Similarly, the sequence for the dash function is turn on LED, wait 600ms, turn off LED and wait 200ms. Note that, for simplicity, the time needed between consecutive dashes and dots is included, which implies that the time between letters will be 400ms.

  2. S.O.S.

    Following the ideas of the previous exercise, implement two functions that accepts the number of repetitions on each of the dot and dash functions. Now, instead of make the LEDs blink, make the sound buzzer (connected to pin D5) beep.

    Hint:

    Add an integer parameter and repeat the dash and dot sequence inside a for loop instruction. Now, the sound buzzer must wait the base time for dots and three times the base time for dashes. In both base, the waiting time once has stop beeping is the base time. Use a tone of 1500Hz for the beep.

  3. Encrypt

    Implement a function that accepts a string message (as input argument) and returns a string. The function must encrypt a message based on the Atbash encryption table.

    Hint:

    Inside the function, implement a while loop, that iterates over the characters of the string. Then perform new_c←'z'-c+'a', being c the read character and new_c the character to append to the output string.
