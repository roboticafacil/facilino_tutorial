USB Serial Instructions

Learn how to print to the console and read from the console with the USB serial interface.

Required material.
  • An Arduino or ESP board, i.e.: Arduino Uno, Arduino Nano, Wemos D1 R2, Wemos D1 R32, NodeMCU, etc...
  • A Multisensor shield, a potentiometer connected to pin A0 or any analog sensor to play around.
USB Serial Monitor

Arduino and ESP boards can be programmed using the USB port. However this is not the only purpose for this port, because it can be used to transmit data between your PC and the board. You can transmit any data from your board using 'Print' instructions in Facilino or send data from your PC and read and use that data in your board using 'Read' instructions. In order to do so, we use the Serial Monitor, this is an integrated console application inside Facilino. You can also use Arduino IDE monitor application or any other serial console application that suits you.

The aim of a serial monitor (or console) application is to display all characters received from a serial port and to provide means for sending data. In order to succeed with that, you usually need to setup your baudrate (speed at which data is transmitted) and other communication parameters. This might depend on your board configuration. Facilino sets the default baurdrate for you at the beginning of your code whenever you use a USB serial instruction and this depends on the selected hardware. For Arduino boards, the default baudrate is 9600bps (bits per second). This might quite fast, but indeed is not one of the fastest ones. ESP processors transmit data through the serial port at 115200bps! Facilino Monitor configures this speed automatically for you depending on the selected hardware, but if you use Arduino IDE or any other serial monitor, you will need to properly configure this value.

Once you click on the 'Monitor' icon in Facilino, it will automatically open the serial port and start displaying whatever data is send. In order send data, you need to write on the top text input and press enter. You can also show graph plots if you use the 'Graph' icon on the top left corner. This will allow you to monitor data varying over the time. This is still a feature that needs to be improved, so maybe, it is easier for you to understand plot data if you use the Arduino IDE Serial Plotter.

Basic Exercises
  1. Hello world!

    At the start up, print the sentence "Hello world!" on the console.

    Hint:

    Use the 'Print Line' instruction and add the text with the sentence "Hello world!".

     
  2. Hello world again!

    Now, print the sentence "Hello world!" on the console at the start up and then the sentence "Hello world again!" in the main loop.

    Hint:

    Use the 'wait' instruction to give some time after using the 'Print Line' instruction.

     
  3. Read potentiometer values

    Create a program that sends over the serial port data measured with the potentiometer (connected to pin A0).

    Hint:

    Use the 'Print Line' instruction, but new instead of using text, you can use the values returned from the 'AnalogRead' instruction. You can use a 'Wait' instruction to control the frequency the values are printed (i.e.: 100ms).

     
  4. Serial loopback

    Create a program that sends over the serial port data received on it (loopback).

    Hint:

    If there's data available on the serial port, then use the 'Read String' instruction and send it back with the 'Print Line' instruction.

     
  5. Plot a signal.

    Using Arduino's plotter, plot the analog value of the potentiometer (connected to pin A0).

    Hint:

    Use the 'plot data' instruction to plot the value return by the 'AnalogRead' instruction and select the pin A0 with the 'Analog PIN' instruction. Use a delay of 50ms with the 'Wait (ms)' instruction, so that the samples are spaced at that sampling period.

Intermediate Exercises
  1. Parsing integer numbers

    Create a program that read from the serial USB port an integer number. Once you read the number add one to it and print it back to the console.

    Hint:

    Use the 'read integer' instruction instead the 'Read String'. Use the 'math arithmetic' instruction to add one to the number.

     
  2. Parsing float numbers

    Modify the previous program so that it expects a float number. The difference is that now, we can use it to perform floating point operations. For instance, you can use it to print on the console its negate value.

    Hint:

    Use the 'Read Float' instruction and the negation instruction (is one of the listed options in the 'floating point operations' (by default is shown 'square root' as first option).
