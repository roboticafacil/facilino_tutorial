Sound Buzzer

Learn how to generate predefined sounds, melodies and even melodies that run in a background task.

Required material.
  • An Arduino or ESP board, i.e.: Arduino Uno, Arduino Nano, Wemos D1 R2 or Wemos D1 R32, NodeMCU, etc...
  • A Multisensor shield or two push-buttons connected to pins D2 and D3 and a buzzer connected to pin D5.

Basic Exercises
  1. Playing with notes

    Reproduce some simple music notes, i.e.: play all notes from A4 to C5 with a quarter duration at the setup.

    Hint:

    Use the 'Buzzer' instruction.

     
  2. Combining notes and no tones

    Now, use the 'No Tone' instruction just after each note and see the difference.

     
  3. Playing with notes with ties

    Now, instead of using the 'No Tone' instruction check the 'tie' option and see the difference.

    Hint:

    When activating the 'tie' option, notes are not interrupted before playing the next note.

Intermediate Exercises
  1. Predefined Sounds

    Generate a connection sound when the switch button D2 is pressed and a disconnection sound when released. The Sound created is up to you.

    Hint: 

    Predetermine bolean variables, that remember the state the button is in. Always add a little delay of your loop because sensors and actuaters need some time to react.

     
  2. Custom Melodies

    Generate the melody "Close Encounters ot the Third Kind". Wait for the Button D2 to start reproducing the melody.

    Hint: 

    Use the individual notes to create the customized melody.

  3. Predefined Melodies on a Background Task

    Generate a predefined melody on a backgroung task that can be interrupted anytime. The melody will start on pushing the switch button D2 and interrupted when switch button D3 is pressed.
