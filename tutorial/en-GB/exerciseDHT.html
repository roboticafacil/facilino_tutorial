Digital Temperature and Humidity (DHT11)

Learn how to read temperature and humidity data from Facilino and set an alarm if temperatures or humidities are too high or too low.

Required material.
  • An Arduino or ESP board, i.e.: Arduino Uno, Arduino Nano, Wemos D1 R2 or Wemos D1 R32, NodeMCU, etc...
  • A Multisensor shield or a DHT11 sensor module.

Basic Exercises
  1. Read Temperature

    Read tempeperature with the DHT11 sensor connected to pin D4 every 5 seconds and print the result in the serial monitor.

    Hint:

    Use the 'Temperature sensor' and 'Wait (s)' instructions.

     
  2. Read Humidity

    Now, read humidity with the DHT11 sensor connected to pin D4 every 5 seconds and print the result in the serial monitor

    Hint:

    Use the 'Humidity sensor' and 'Wait (s)' instructions.

     
  3. Read Temperature every 5000ms

    Now, modified the first exercise and read the temperature every 5000ms using the 'Every' instruction. Also include a text to print together with the temperature value: 'Temperature (Celsius): '

    Hint:

    Use the 'Temperature sensor' and 'Every' instructions.

     
  4. Read Humidity every 5000ms

    Now, modified the first exercise and read the humidity every 5000ms using the 'Every' instruction. Also include a text to print together with the humidity value: 'Humidity (%): '

    Hint:

    Use the 'Humidity sensor' and 'Every' instructions.

     
  5. Temperature Alarm

    Print a message if the temperature is too hot or too cold

    Hint:

    Use the 'Temperature Alarm' instruction. There's no need to include an extra delay after the alarm instructions because internally waits 1000ms after the checking the alarm cases.

  6. Humidity Alarm

    Print a message if the humidity is too humid or too dry

    Hint:

    Use the 'Humidity Alarm' instruction. There's no need to include an extra delay after the alarm instructions because internally waits 1000ms after the checking the alarm cases.

Intermediate Exercises
  1. Read Temperature and Humidity every 5000ms

    Now read both, temperature and humidity, every 5000ms. .

    Hint:

    Now, declare two variables 'temperature' and 'humidity' and store the result of reading both values. Include a delay of 1000ms between temperature and humidity readings to ensure that the sensor works as expected.

     
  2. Plotting Temperature & Humidity

    Plot temperature and humidity every 5000ms and show it in the console. Use the DHT11 sensor.

     
  3. Temperature & Humidity Alarms

    Print mesagges on the console when the temperature and humidity changes to unconfortable values, including 'too hot', 'perfect temperature', 'too cold', 'too humid', 'perfect humidity' and 'too dry' cases.

    Hint:

    Use Temperature and Humidity alarm instructions. There's no need to include an extra delay between two consecutive alarm instructions because internally waits 1000ms after the checking the alarm cases.
