Logic

Learn how to work with logic instructions with Facilino, including Boolean and bitwise operations.

Required material.
  • An Arduino or ESP board, i.e.: Arduino Uno, Arduino Nano, Wemos D1 R2 or Wemos D1 R32, NodeMCU, etc...
  • A Multisensor shield or two LEDs (red and blue connected to pins D12 and D13) and two push-buttons connected to pins D2 and D3.
Boolean Logic
As you know, binary digital signals can only take two possible values 0V or 5V. A binary variable is very similar to that, because it can only take two possible values TRUE or FALSE, usually denoted as 1 or 0. Boolean algebra is the part of albegra working with binary variables, that is the rules and symbols for working with binary variables. It was formalised by George Boole, introducing the logic relations describing the equivalent relations as in numeric algebra.
There are three basic operations: AND (conjuntion), OR (disjunction) and NOT (negation). An AND operation accepts two values and returns 1 if, and only if, both of them are 1. An OR operation accepts two values and returns 1 whenever any of them is 1. A NOT operation accepts only one value and returns the opposite of the input, that is 0 if it is 1 or 1 if it is 0 (clearly the negation of a negated value is equivalent to the original value). There are other useful operations, like the XOR or exclusive OR. This is an operation that takes two binary digits and returns 0 if they are the same and 1 if they are different.

In addition to this, there are identities when combining these basic operations, that are known as De Morgan's Law:

X AND Y=NOT(NOT(X) OR NOT(Y))

X OR Y=NOT(NOT(X) AND NOT(Y))

Bits and Numbers
Computers don't understand numbers the way we do. Neither microcontrollers such as Arduino and ESP. Instead, they do understand bits. A bit is the basic unit of information. As you may know, a bit only is able to store two values or states: 0 or 1. So in order to represent bigger numbers with need groups of bits, in the same way we use decimal representation with digits from 0 to 9, every time we need to represent a bigger number, let's say bigger than 9, we increase a digit on the left side using 1 and set the rest of digits to 0, that's it, we create 10. In binary representation it is exactly the same way, we need to increase with a new digit on the left side with a 1 and set all of other digits with 0s. Therefore, we can represent the following decimal numbers with an equivalent binary representation (we have included a small 'b' to denote the binary representation):

In microcontrollers, when we use a number it is usually stored in memory and that requires a given amount of bits. Typically, we store data in groups of 8-bit representations (byte), 16-bit representation (word or int) or 32-bit representation (double word or long int). Therefore, a byte can store upto 2^8 numbers, that is from 0 to 255. A word, can represent numbers from 0 to 65535 and a double word can represent from 0 to 4294967295. Yes, this might be big, but sometimes we need even more. Sometimes we need to represent negative numbers. The convention here is to use the most significant bit or MSB (the bit on the left) to represent the negation sign.
Bitwise operations
Now, you know a little bit about numbers and bits, you might be interested in knowing that bitwise operations can be done with numbers. This might be useful to check whether or not a bit is 1 or 0, because this bit is representing some kind of useful information for a given data. Yes, we know this is going to deep on how data is represented, but believe us when we say that there are many applications and reasons for that. Just an example, an easy way to check if your data being transmitted propperly is to use an error check system. One of the simplest ways of doing that is to perform an XOR operation on sent data and then send a last byte with the result of this XOR operation. When the receiver receives the data, uses the same XOR operation on data and if the resuts match, then it is very likely that the data is correct and has not been corrupted.

Basic Exercises
  1. Set LED when pushed

    Following the ideas of basic exercise 3 of Set LED when not pushed, we want to set the LED connected to pin D12 when we push the push-button (connected to pin D2).

    Hint:

    Use the 'DigitalRead' instruction to get the value and its returned value to the 'not' instruction to negate its value. The resulting value must be used as the state input of the 'DigitalWrite' instruction.

  2. AND operation

    Implement Boolean operation using the two push-buttons (connected to pins D2 and D3) and the red LED (connected to pin D12). When both of them are pushed, the pin of the LED must be set to HIGH.

    Hint:

    Considering push-button D2 as PB1 and push-button D3 as PB2, the resulting operation for setting the LED state is LED=NOT(PB1) AND NOT(PB2).

  3. Equivalent AND operation

    Now, using De Morgan's Law, find an equivalent implementation of previous exercise but with less operations.

    Hint:

    In this case, you need to implement LED=NOT(PB1 OR PB2).

  4. Simple comparison

    Set the LED connected to pin D12 to HIGH when the value read from the analog pin A0 (potentiometer) is higher than 512.

    Hint:

    Use the '>' instruction to compare the number return by the instruction 'AnalogRead' and the constant number 512. The result of this operation can be used as input decision of an 'If' instruction to decide whether or not to set the LED.

Intermediate Exercises
  1. AND and OR Operations

    Implement binary operations using the button switches (connected to pins D2 and D3) and the red and blue LEDs (connected to pin D12 and D13). The aim is to implement the following logic operations: RED=NO(PB1 AND PB2), BLUE=NO(PB1 OR PB2).

  2. Even or Odd (Bitwise And)

    Turn on LED D12 whenever we introduce an even number on the USB serial console. Turn on LED D13 when it's odd.

    Hint:

    Use a bitwise 'and' to mask the number with '1'. If the result is greater than zero then the number is odd. To parse an integer number Intermediate Exercise 1 in USB Serial Communication.

  3. Comparison

    Read the analog input A0 corresponding to the potentiometer position. Divide the analog read range into 4 sub-ranges and turn on LEDs D12 and D13 based on the binary codification of each range.

    Hint:

    If the number is between 0 and 1023, then both LEDs must be off, if the value is between 1024 and 2047 then LED D12 must be on and D13 off, if the value is between 2048 and 3071 then LED D13 must be on and D12 off. Otherwise, both LEDs must be on. Remember that for ESP processor the range for analog inputs is from 0 to 4096, while if you use Arduino, the range is from 0 to 1023.
