Variables

Learn how to work with local and global variables and their types. Also learn how to work with array variables.

Required material.
  • An Arduino or ESP board, i.e.: Arduino Uno, Arduino Nano, Wemos D1 R2 or Wemos D1 R32, NodeMCU, etc...
  • A Multisensor shield or two LEDs (red and blue connected to pins D12 and D13), a potentiometer connected to A0, a LDR connected to pin A1 and a temperature sensor LM35 connected to pin A2.
Variable's scope

When we put something in a box, we can take it out at anytime. Variables are just like boxes, we use them to store data of our program and we can read that data at anytime. But there's one important thing to consider: we can only take something out of a box or put something in a box if the box is within our scope, i.e.: if we can reach the box. Similary, variables do have a scope: they can be local or global. Global variables are always accessible, while local variables are only accessible within the variable's scope. Therefore, a global variable is like a box that we carry with us always, but local variables are like boxes that we store them in a room, and can only be accessed if we are inside that room, but not from outside.

The variable scope is delimited to the context where it has been declared. In Facilino, any statement input accepting code instructions is indeed a context for a variable. For instance, the 'setup/loop' instructions has two contexts, the 'setup' context and the 'loop' context. A 'for' loop has its own context inside the loop, and so on...

Variable types

While for a person it is easy to interpret Variables or Symbols in their context, programms need pretermined "Types" to correctly use them. First of all you have a simple boolean, that can only take the values true/1 or false/0. You have full numbers, that vary in maximal and minimal value; byte, int & long int. You have decimal numbers, that also include full numbers; float. You have alphabetical characters and extra symbols in a char. For a full word or a complete sentence, you have to combine multiple chars to a string.

Array Variables

When we need to store large amount of data, it is not practical to create thousand and thousands of variables, each of them with a different name to store that data. It is much more practical to store data in an array, under one single name. Each of the elements of an array can be accessed individually, the important aspect here is that each element is ordered based on each position index, starting from 0. In Facilino, we can create array data, but it must be statically declared in advance, i.e. the array size much be known and does not change. In order to declare an array size, we use an array construction, which specifies the amount of data that the array variable can store.

The Chessboard Legend
The rice and the chessboard story. There's a famous legend about the origin of chess that goes like this. When the inventor of the game showed it to the emperor of India, the emperor was so impressed by the new game, that he said to the man "Name your reward!"
"Oh emperor, my wishes are simple. I only wish for this. Give me one grain of rice for the first square of the chessboard, two grains for the next square, four for the next, eight for the next and so on for all 64 squares, with each square having double the number of grains as the square before."
The emperor agreed, amazed that the man had asked for such a small reward - or so he thought. After a week, his treasurer came back and informed him that the reward would add up to an astronomical sum, far greater than all the rice that could conceivably be produced in many many centuries!

Basic Exercises
  1. Local variable

    Implement a program that prints the value of a local variable at the 'setup' context and prints the value of another local variable (with the same name) in the 'loop' context.

    Hint:

    Create a local variable inside the 'setup' context and assign a value to it and print its value. Then, create another local variable with the same name and assign a different value. You will see that this does not produces a compilation error..

     
  2. Global variable

    Now, create a program that creates a global variable in the 'setup' and uses its value in the 'loop' section.

    Hint:

    Now, use the instructions with the GLOBAL keyword, to declare a global variable. The global variable only has to be declared once, normally in the 'setup'.

     
  3. Variable context

    Create a global variable with name 'iteration' (in the 'setup' section) and initialize its value to 5. In the main loop, make the red LED connected to pin D12 blink 10 times using a 'for' loop. Inside that 'for' loop, declare a local variable with the name 'iteration' and assign the value of the current iteration (1, 2, ..., 10). After, LED D12 has blinked 10 times, now make the blue LED connected to pin D13 blink as many times as the value of the variable 'iteration' using another 'for' loop. How many times does the blue LED blink?

    Hint:

    The blue LED will only blink five times because the local variable only has a valid context within the 'for' loop, and thus, outside the 'for' loop, the global value with used. You can print the value of the variable just before, inside and after the 'for' loop.

     
  4. Moving average

    Following the ideas of Basic Exercise 4 in Functions and procedures, implement a moving average with values from the potentiometer connected to pin A0. The difference now, with respect to Basic Exercise 4 in Functions and procedures, is that only one measurement is read on each iteration. To simplify, we use the last 10 measurements stored in an array to compute the mean. On each iteration the oldest element of the array will leave out, while only one single measurement will be added at the beginning of the array. The moving mean implies that the previous mean can be reused and only the oldest and newest values in the array are needed in order to compute the mean, see Moving average.

    Hint:

    Create an array with 10 initial values (type float). Create also a floating point variable with the previous mean, initialized with a zero. The oldest value will be the element on the position 9, while the newest element will be the one with the position 0. In the main loop, first you keep the value of the oldest element, that is, the element at position 9; second, you need to shift the values of the array so that the value of the element at position 8 is stored at position 9... until element in position 0 is stored in position 1; third, you can measure the value of the potentiometer and store its value at position 0; fourth, you can compute the moving average using the previous mean value.

Intermediate Exercises
  1. How many rice grains in total?

    Compute the requested number of grains. Use different number representations to see which can represent the number. Show on the console the total amount computed when using integers, long integers and floats. All required Electronics are already integrated in the Multisensorshield.

    Hint:

    Implement a 'for' loop with 64 iterations and compute the powers of 2 (declare variables of appropriate type and initialize them to 1). At the end, print the results to see if they are correct.

     
  2. Send Array Data

    Encode readings of analog inputs A0, A1 and A2 and send data over the USB serial.

    Hint:

    Build an array data of type 'short integer' with three elements and store each of the analog readings on scalar 'short integer' variables. Then, encode them on the array data and send them over the USB serial port.

     
  3. Store Array Data

    Decode data received from the USB serial port with byte values of three color channels. Use received data to regulate the color of RGB (pins D9-D11-D10, respectively).

    Hint:

    If there's data available on the serial port, declare an array data of type 'byte' with three elements with the read byte data from the USB serial. Access to each of the elements of the data to set the 'analog write' value for each LED channel.
