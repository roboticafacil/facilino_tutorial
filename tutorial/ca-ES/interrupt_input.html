Entrada de Interrupción

Una descripción detallada de las entradas de interrupción de un Arduino

Interrupciones

Las interrupciones son ciertas señales que interrumpirán el flujo del programa actual para que se pueda ejecutar una tarea de mayor prioridad o una interrupción. Los programas de microcontroladores funcionan normalmente en un orden lineal, que se repite una vez finalizado. Generalmente, la mayoría de los microcontroladores incluyen diferentes tipos de interrupciones para permitir al usuario saltar entre tareas para tener un flujo de trabajo más útil y natural.

Temporizador

Los temporizadores se incluyen en la mayoría de los microcontroladores para medir el tiempo o ejecutar tareas periódicamente. Los temporizadores incluyen registros donde almacenar el valor del contador del temporizador actual. Este valor se actualiza con la señal de reloj del microcontrolador junto con un prescaler (para hacer que la señal de actualización sea más lenta, porque los tics del reloj suelen ser demasiado rápidos). Cuando el contador del temporizador alcanza un valor predefinido, esto activa una interrupción para que se pueda ejecutar una tarea específica. Para controlar el período en el que se ejecuta la interrupción, se debe modificar el valor predefinido.

Interrupciones Externas

Las interrupciones externas pueden generar una interrupción en el flujo de trabajo en pines específicos. La interrupción se puede activar cuando el pin asosciado a la interrupción tiene un nivel determinado (ALTO o BAJO) o en los flancos ascendente o descendente de la señal, es decir, las transiciones de ALTO a BAJO (flanco descendente) o BAJO a ALTO (flanco ascendente).

Instrucciones de Facilino

Para generar interrupciones basadas en temporizador, Facilino utiliza una instrucción de tareas periódicas que te permite ejecutar tareas periódicamente con un período mínimo de 1 ms utilizando la primera instrucción. Esta instrucción es una instrucción mutable, lo que significa que puedes agregar tantas tareas como desees. Precaución: cuando trabajes con temporizadores, debes tener en cuenta el hecho de que otras bibliotecas también podrían usar ese temporizador, por lo que la ejecución de tareas que requieren mucho tiempo dentro de esta instrucción podría generar muchos problemas y las cosas no funcionarán como se espera. Este es un problema común surgido debido a compartir un recurso común y Facilino hace todo lo posible para superar parte de estas situaciones y contempla casos en las que el código generado funciona como uno espera, pero puede haber casos en los que pueda haber problemas.

La segunda instrucción se utiliza para ejecutar el código en el modo de señal de interrupción indicado (bajo, cambio, flanco ascendente o flanco descendente). Nuevamente, el uso de este tipo de instrucción podría afectar a la ejecución de otras instrucciones, debido a la ejecución del código dentro de la instrucción. Idealmente, este código debería ser rápido de ejecutar.
